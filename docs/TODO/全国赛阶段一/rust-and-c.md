#  Rust 与 C 类型关系

|   C 类型    | 对应的 Rust类型 | 在 RV32 的字节数 | 在 RV64 的字节数 |
| :---------: | :-------------: | :--------------: | :--------------: |
|    char     |       i8        |        1         |        1         |
|    short    |       i16       |        2         |        2         |
|     int     |       i32       |        4         |        4         |
|  **long**   |    **isize**    |      **4**       |      **8**       |
|  long long  |       i64       |        8         |        8         |
|    void*    |      usize      |        4         |        8         |
|    float    |       f32       |        4         |        4         |
|   double    |       f64       |        8         |        8         |
| long double |        -        |        16        |        16        |

举个因为类型解析出错的例子：

在 c 语言系统调用某个参数为 `int*`类型。rust 按 `*mut isize` 解析时，由于 c语言 int 解指针只截取了 rust isize 的前半段，导致结果为 0 而不是预期结果。
